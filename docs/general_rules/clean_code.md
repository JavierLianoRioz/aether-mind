# Reglas de Clean Code

Este proyecto se adhiere estrictamente a los principios de Clean Code para asegurar la mantenibilidad, legibilidad y escalabilidad del código.

## Principios Fundamentales

1.  **Código Auto-documentado**: El código debe ser claro y explicativo por sí mismo. Evita comentarios innecesarios que expliquen "qué" hace el código; enfócate en nombres de variables y funciones descriptivos.
2.  **Responsabilidad Única (SRP)**: Cada función, clase o módulo debe tener una única razón para cambiar.
3.  **No te Repitas (DRY)**: Evita la duplicación de lógica. Abstrae el código común en funciones o clases reutilizables.
4.  **Mantenlo Simple (KISS)**: La solución más simple suele ser la mejor. Evita la sobreingeniería.
5.  **Nombres Significativos**: Usa nombres claros, pronunciables y que revelen la intención para variables, funciones y clases.
6.  **Funciones Pequeñas**: Las funciones deben ser pequeñas y hacer una sola cosa.
